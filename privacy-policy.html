Privacy Policy

YouTube Comment Overlay Chrome Extension  ·  Last updated: 2026-06-18

1. Overview

YouTube Comment Overlay ("the Extension") displays YouTube video comments as an overlay on the video player. This policy explains what data the Extension accesses and how it is handled.

2. Data We Do NOT Collect

The Extension does not:

  • Collect, store, or transmit any personally identifiable information.
  • Track browsing history or viewing habits.
  • Send any data to our servers — we operate no backend servers.
  • Use cookies or any form of cross-site tracking.

3. YouTube Data API

The Extension uses the YouTube Data API v3 to fetch public comment data for the currently playing video. This API is provided by Google LLC and is subject to Google's Privacy Policy.

API requests are made directly from your browser to Google's servers (https://www.googleapis.com/youtube/v3/). No comment data is stored beyond your current browser session.

4. API Key

The Extension requires a YouTube Data API key provided by the user. This key is stored locally in Chrome's chrome.storage.sync and is only used to authenticate requests to the YouTube Data API. It is never transmitted to any third party other than Google.

5. Permissions

  • storage — saves your settings (API key, display mode, position, opacity, language) locally.
  • activeTab — injects the comment overlay into the currently active YouTube tab.
  • https://www.youtube.com/* — required to detect the currently playing video.
  • https://www.googleapis.com/youtube/v3/* — required to fetch public comment data via the YouTube Data API.

6. Children's Privacy

The Extension does not knowingly collect any information from children under the age of 13.

7. Changes to This Policy

We may update this Privacy Policy from time to time. The "Last updated" date at the top of this page will reflect any changes. Continued use of the Extension after changes constitutes acceptance of the updated policy.

8. Contact

If you have any questions about this Privacy Policy, please open an issue on our GitHub repository.
